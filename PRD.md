# PRD — VibeCoding for Aisulu

**Тип документа:** Product Requirements Document
**Проект:** Лендинг-платформа «VibeCoding for Aisulu»
**Версия:** Vol. I — MMXXVI
**Формат:** Одностраничный интерактивный сайт (HTML/CSS/JS, без фреймворков)
**Файл сайта:** `index.html`

---

## 1. Обзор проекта

**VibeCoding for Aisulu** — обучающий сайт для новичков, которые никогда не занимались вайбкодингом. Сайт объясняет три базовые темы: как писать промпты, какими инструментами пользоваться, и как выпустить свой первый проект — плюс отдельный блок с короткими практическими советами (tips).

Ключевое требование к продукту — не выглядеть как типичный SaaS/edtech-лендинг. Визуальный язык — люксовый fashion-эдиториал (Vogue / Saint Laurent / Prada / Kinfolk): чёрно-бордовая палитра, крупная эдиториальная типографика, минимализм, сдержанная анимация.

**Формула проекта:** *"Vogue встречает софтверный инжиниринг"* — контент для начинающих, но подан как эксклюзивное модное издание.

---

## 2. Цели продукта

- Научить абсолютного новичка трём базовым навыкам вайбкодинга: промптинг, выбор инструментов, выпуск первого проекта.
- Дать быстрые практические советы (tips), не привязанные к формальному уроку.
- Создать ощущение эксклюзивности и премиальности — чтобы обучающий сайт не считывался как курс/бутлкемп.
- Сделать навигацию нестандартной и запоминающейся, а не шаблонным скроллом.

## 3. Целевая аудитория

Новички, которые:
- не умеют программировать и не изучали синтаксис;
- хотят создавать продукты через AI-инструменты (Claude, Cursor, деплой-сервисы);
- ценят эстетику и не хотят «корпоративного» обучающего интерфейса.

---

## 4. Архитектура навигации

Сайт построен как **SPA с орбитальной моделью навигации**, а не как обычный скролл-лендинг.

### 4.1 Модель
- Центральный экран — большой круг с названием сайта в центре.
- Вокруг центра по окружности расположены 6 точек-кнопок (узлов), каждая соответствует одному разделу.
- **Наведение (hover)** на узел — не переходит никуда, а показывает краткое описание раздела (label + brief) прямо под названием сайта в центре круга.
- **Клик** на узел — плавный переход (fade + blur + scale) на полноэкранный блок этого раздела.
- Внутри каждого раздела есть кнопка **«← Back to Constellation»**, возвращающая на орбиту.
- **Скролл между разделами отключён** — переключение экранов возможно только кликом по кнопке (узлу орбиты или Back-кнопке). Скролл разрешён только *внутри* открытого раздела, если контента больше экрана.

### 4.2 Разделы (узлы орбиты)

| № | Узел | Назначение |
|---|------|-----------|
| 1 | **Manifesto** | Философия: почему вайбкодинг начинается с интуиции, а не с синтаксиса |
| 2 | **Chapter I — The Prompt** | Как писать промпты: контекст, точность, итерации, референсы |
| 3 | **Chapter II — The Instruments** | Три инструмента: Claude, Cursor, Netlify |
| 4 | **Chapter III — The First Project** | Как выбрать и выпустить свой первый проект |
| 5 | **Studio Notes** | 6 коротких практических tips |
| 6 | **Enroll** | Финальный призыв, возврат на орбиту |

## 5. Дизайн-система

### 5.1 Палитра (token system)

| Название | HEX | Назначение |
|---|---|---|
| `--black` | `#1B1815` | Основной фон (тёплый графит, светлее исходного чёрного) |
| `--charcoal` | `#242019` | Вторичный фон, секции |
| `--graphite` | `#2B2620` | Hover-состояния карточек |
| `--cherry` | `#7A1024` | Акцент, выделение (::selection) |
| `--burgundy` | `#96182F` | Основной акцентный цвет (курсив, hover, точки орбиты) |
| `--warm-white` | `#F5F4F2` | Основной текст |
| `--silver` | `#B3ABA1` | Второстепенный текст |
| `--champagne` | `#D8CBB4` | Мелкие лейблы, eyebrow-текст |
| `--hairline` | `rgba(245,244,242,0.16)` | Тонкие разделители/границы |

Правила палитры: без ярких/насыщенных цветов, без градиентов «в стиле tech», без синего/фиолетового/неона.

### 5.2 Типографика

- **Display / заголовки:** `Fraunces` (serif, editorial, с курсивным начертанием для акцентных слов)
- **Текст / интерфейс:** `Inter` (sans-serif, светлые начертания 300–500)
- **Все заголовки (h1–h4, логотип, цитаты) — капс** (`text-transform: uppercase`)
- Обычные параграфы и описания — обычным регистром, без капса
- Эффект: крупные заголовки + мелкие uppercase-лейблы (eyebrow) с трекингом (letter-spacing 0.2–0.32em)

### 5.3 Слои и структура компонентов

- Числа глав (I / II / III) — крупные, обводка без заливки (`-webkit-text-stroke`), декоративные
- Инструменты и Notes — карточки на общей сетке с hairline-границами (1px), без теней/скруглений
- «Plates» — декоративные графические блоки (замена фотографий) с параллаксом при скролле внутри раздела, зернистая чёрно-белая текстура через градиенты

### 5.4 Моушн/интеракции

- Кастомный курсор (точка + кольцо, `mix-blend-mode: difference`), увеличивается при наведении на интерактивные элементы
- Film grain оверлей (SVG `feTurbulence`, `mix-blend-mode: overlay`, opacity ~0.045)
- Переходы между экранами: `opacity + blur + scale`, ease `cubic-bezier(0.16, 1, 0.3, 1)`, 850мс
- Вращающееся декоративное кольцо вокруг орбиты (100s linear, бесконечно)
- `prefers-reduced-motion` уважается — анимации отключаются

## 6. Контент сайта (копирайт)

### 6.1 Шапка / логотип
`VIBECODING for AISULU` — «Vol. I — MMXXVI»

### 6.2 Центр орбиты
- Eyebrow: *An Editorial in Code*
- Заголовок: *VibeCoding for Aisulu*
- Дефолтный тэглайн: *Six Points, One Instinct — Hover a mark to preview. Click to enter the chapter.*
- При наведении на узел тэглайн подменяется на `data-label` + `data-brief` этого узла

### 6.3 Manifesto
- Цитата: *"You do not need to know how to code. You need to know how to ask."*
- Абзацы: про вайбкодинг как направление интеллекта через язык, а не написание синтаксиса; про то, что это курс для новичков — 3 главы, без пререквизитов

### 6.4 Chapter I — The Prompt
Заголовок: *The Prompt is your voice.*
Лид: промпт — это бриф, а не поисковый запрос.
4 шага:
1. Give context first, instructions second
2. Precision reads as confidence
3. Iterate like an editor, not a critic
4. Show, don't just tell

### 6.5 Chapter II — The Instruments
Заголовок: *The Instruments — chosen like fabric.*
3 инструмента:
- **Claude** — «The Conversation» (где происходит мышление и написание первой версии)
- **Cursor** — «The Workshop» (редактор для точного управления кодом)
- **Netlify** — «The Runway» (деплой — выход проекта в мир)

### 6.6 Chapter III — The First Project
Заголовок: *The First Project — released, not finished.*
4 шага:
1. Choose something small enough to finish this week
2. Describe the whole thing before you build any of it
3. Build in public view, even if the audience is only you
4. Release before it feels ready

### 6.7 Studio Notes (6 tips)
1. Описывай, каким код должен стать — не то, чего избегать
2. Расплывчатый промпт — черновик, а не провал; перепиши перед оценкой результата
3. Сохраняй рабочую версию перед тем как пробовать рискованную
4. Пиши промпт как инструкцию доверенному ассистенту, а не как поисковый запрос
5. Если результат не тот — не так была сформулирована задача, а не «руки не оттуда»
6. Веди архив рабочих промптов — он ценнее, чем библиотека синтаксиса

### 6.8 Enroll (закрытие)
Заголовок: *Enrollment is by instinct, not application.*
Кнопка возврата на орбиту.

## 7. Технические требования

- **Стек:** чистый HTML + CSS + Vanilla JS, один файл `index.html`, без сборки и зависимостей (кроме Google Fonts CDN: Fraunces, Inter)
- **Навигация:** JS-переключение `.view` контейнеров через `classList` (`active`), без роутинга/фреймворка
- **Скролл:** `html, body { overflow: hidden }` — глобальный скролл отключён; `.view { overflow-y: auto }` — скролл разрешён только внутри активного раздела
- **Адаптивность:**
  - Desktop: круговая орбита с узлами, вычисленными через `rotate()` + `translateX()` + обратный `rotate()`
  - Mobile (≤760px): орбита не отображается как круг, разбивается на вертикальный список
- **Доступность:** уважение `prefers-reduced-motion`, видимый фокус для интерактивных элементов, `cursor:none` только на устройствах с `hover` (не тач)
- **Производительность:** без внешних изображений (все визуальные «фото» — CSS-градиенты/паттерны), минимальный JS без сторонних библиотек
